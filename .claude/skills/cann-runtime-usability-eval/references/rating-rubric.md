# 文档质量 A/B/C/D/F 等级标准

## 评级定义

### A 级 — 优秀

**标准：** 内容完整、有示例、与代码同步、有交叉引用

具体要求：
- 所有参数/返回值/错误码有完整说明
- 至少一个可执行的代码示例
- 描述与当前版本代码一致
- 链接到相关文档和示例
- 有注意事项和最佳实践
- 格式规范、结构清晰

### B 级 — 良好

**标准：** 内容基本完整，缺示例或部分参数说明不全

具体要求：
- 核心参数有说明，但可能缺少边界条件或可选参数说明
- 可能缺少代码示例，或示例不够完整
- 与代码基本同步，可能有少量偏差
- 格式基本规范

### C 级 — 及格

**标准：** 框架存在但内容不完整，关键信息缺失

具体要求：
- 有基本结构和框架
- 缺少关键参数说明或错误码说明
- 无代码示例
- 可能与代码有较大偏差
- 开发者需要靠猜测才能使用

### D 级 — 不及格

**标准：** 仅有标题或占位内容，无实质信息

具体要求：
- 文件存在但内容几乎为空
- 可能只有标题和占位文字
- 无法从中获取任何有用信息
- 等同于文档缺失

### F 级 — 缺失

**标准：** 完全缺失

具体要求：
- 该类型文档完全不存在
- 在 docs/ 全部文件中均未找到相关内容

## 评级时的注意事项

1. **评级须基于具体证据**，不得凭主观感觉
2. **边界情况倾向低评级**，宁可严格也不放水
3. **记录评级理由**，说明具体缺了什么或哪里做得好
4. **区分"存在但质量差"和"完全缺失"**，两者的整改优先级不同
