# 维度一：仓库基础设施检查项

## 权重：10%

## 检查清单

### 1. README.md

| 检查项 | 说明 | 严重等级 |
|--------|------|---------|
| 项目简介 | 一段话说明项目定位和核心功能 | 阻塞性 |
| 适用人群 | 明确目标用户（算子开发者、框架开发者等） | 影响体验 |
| 快速开始 | 5 分钟内能跑通的最简示例 | 阻塞性 |
| 依赖说明 | 软硬件依赖、版本要求 | 阻塞性 |
| 构建步骤 | 从源码编译的完整步骤 | 影响体验 |
| 目录结构说明 | 仓库各目录的作用 | 优化建议 |
| 徽章/状态标识 | CI 状态、版本号、许可证等 | 优化建议 |

### 2. LICENSE

| 检查项 | 说明 | 严重等级 |
|--------|------|---------|
| 文件存在 | 是否有 LICENSE 文件 | 阻塞性 |
| 协议明确 | 使用的开源协议是否清晰 | 阻塞性 |
| 兼容性 | 与依赖库的协议是否兼容 | 优化建议 |

### 3. CONTRIBUTING.md

| 检查项 | 说明 | 严重等级 |
|--------|------|---------|
| 文件存在 | 是否有贡献指南 | 影响体验 |
| 代码规范 | 编码风格、命名规范 | 优化建议 |
| PR 流程 | 如何提交贡献 | 影响体验 |
| Issue 指南 | 如何报告问题 | 优化建议 |

### 4. CHANGELOG / Release Notes

| 检查项 | 说明 | 严重等级 |
|--------|------|---------|
| 版本历史 | 是否有版本变更记录 | 影响体验 |
| 变更分类 | 是否按 Added/Changed/Fixed 分类 | 优化建议 |
| 兼容性说明 | 破坏性变更是否标注 | 影响体验 |

### 5. Issue / PR 模板

| 检查项 | 说明 | 严重等级 |
|--------|------|---------|
| Issue 模板 | 是否有 bug report / feature request 模板 | 优化建议 |
| PR 模板 | 是否有 PR 描述模板 | 优化建议 |

### 6. CI/CD 配置

| 检查项 | 说明 | 严重等级 |
|--------|------|---------|
| 配置存在 | 是否有 CI/CD 配置文件 | 影响体验 |
| 覆盖范围 | 是否包含编译、测试、代码检查 | 优化建议 |

## 评分标准

| 分数 | 标准 |
|------|------|
| 5 | 所有项齐全且质量高 |
| 4 | 核心项齐全，部分可选项缺失 |
| 3 | 核心项基本存在但质量一般 |
| 2 | 多个核心项缺失 |
| 1 | 仅有最基本的文件，内容空洞 |
