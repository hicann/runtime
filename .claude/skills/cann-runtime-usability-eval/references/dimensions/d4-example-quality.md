# 维度四：example/ 示例可验证性检查项

## 权重：20%

## 覆盖范围检查

示例应覆盖算子开发者的核心使用路径：

| 场景 | 说明 | 严重等级 |
|------|------|---------|
| Device 初始化与释放 | aclInit/aclrtSetDevice/aclrtResetDevice | 阻塞性 |
| Context/Stream 管理 | 创建、使用、销毁 | 阻塞性 |
| 内存操作 | Host/Device 内存分配、拷贝、释放 | 阻塞性 |
| 核函数加载执行 | Binary 加载、Function 获取、Launch | 阻塞性 |
| Event 事件 | 创建、记录、同步 | 影响体验 |
| 错误处理 | 典型错误检查和恢复 | 影响体验 |
| 端到端示例 | 从初始化到完整算子执行的全流程 | 阻塞性 |

## 逐示例评估项

对 example/ 下每个示例目录/文件，评估以下内容：

| 检查项 | 说明 | 严重等级 |
|--------|------|---------|
| README | 有独立的 README 说明示例用途 | 影响体验 |
| 编译指令 | 有明确的编译命令或 CMakeLists.txt | 阻塞性 |
| 依赖说明 | 依赖的库、工具、环境变量是否交代 | 阻塞性 |
| 预期输出 | 有预期输出说明，开发者能判断对错 | 影响体验 |
| 代码注释 | 关键步骤有注释说明 | 影响体验 |
| 独立可运行 | 不依赖其他示例即可编译运行 | 阻塞性 |
| API 签名一致 | 示例代码与当前头文件 API 签名是否匹配，有无编译错误 | 阻塞性 |
| 与 docs 联动 | 引用或被 docs 引用 | 优化建议 |

## 评估表格格式

| 示例路径 | 功能描述 | 有README | 有编译指令 | 有预期输出 | 注释质量 | 与docs交叉引用 | 综合评级 |
|----------|----------|---------|-----------|-----------|---------|---------------|---------|

综合评级标准：
- **A**：所有检查项通过
- **B**：核心项通过，1-2 项缺失
- **C**：有明显缺失但基本可用
- **D**：缺失严重，难以独立使用
- **F**：无法使用

## 评分标准

| 分数 | 标准 |
|------|------|
| 5 | 示例覆盖全面，每个示例独立可运行，有完整文档 |
| 4 | 核心场景有示例，大部分可运行，部分缺文档 |
| 3 | 有基本示例但覆盖不全，部分示例缺编译说明 |
| 2 | 示例很少或质量差，多数无法独立使用 |
| 1 | 几乎无有效示例 |
