# 维度二：API 接口可用性检查项

## 权重：30%

## 必查接口清单

### Device 管理

| API | 检查项 | 说明 |
|-----|--------|------|
| aclrtSetDevice | 函数签名、参数说明、错误码、示例 | 设置当前设备 |
| aclrtGetDevice | 函数签名、参数说明、错误码、示例 | 获取当前设备 |
| aclrtResetDevice | 函数签名、参数说明、错误码、示例 | 重置设备 |

### Context 管理

| API | 检查项 | 说明 |
|-----|--------|------|
| aclrtCreateContext | 函数签名、参数说明、错误码、示例 | 创建 Context |
| aclrtDestroyContext | 函数签名、参数说明、错误码、示例 | 销毁 Context |

### Stream 管理

| API | 检查项 | 说明 |
|-----|--------|------|
| aclrtCreateStream | 函数签名、参数说明、错误码、示例 | 创建 Stream |
| aclrtDestroyStream | 函数签名、参数说明、错误码、示例 | 销毁 Stream |
| aclrtSynchronizeStream | 函数签名、参数说明、错误码、示例 | 同步等待 |

### 内存管理

### 事件管理

## 补充接口发现流程

1. 扫描 `include/` 目录下所有 `.h` 文件
2. 提取所有 `acl` 前缀的函数声明
3. 与必查清单对比，找出额外暴露的 API
4. 对额外 API 同样进行文档覆盖度检查
5. **在报告中单独列出"仓库额外提供但未在预期清单中的 API"及其文档覆盖情况**

## 每类接口的详细检查项

| 检查项 | 说明 | 严重等级 |
|--------|------|---------|
| 头文件暴露 | API 在 include/ 中可找到 | 阻塞性 |
| 函数签名完整 | 参数名、类型、返回值均有 | 阻塞性 |
| 参数说明 | 每个参数的含义、取值范围 | 阻塞性 |
| 返回值说明 | 成功/失败的返回值 | 阻塞性 |
| 错误码枚举 | 可能返回的错误码列表 | 影响体验 |
| 使用约束 | 线程安全性、调用顺序、前置条件 | 影响体验 |
| 示例代码 | 至少一个可运行的使用示例 | 影响体验 |
| ABI 兼容性 | 版本间接口稳定性承诺 | 优化建议 |

## 评分标准

| 分数 | 标准 |
|------|------|
| 5 | 所有 API 有完整文档 + 示例 + 错误码 + 线程安全说明 |
| 4 | 核心 API 文档完整，少数 API 缺示例 |
| 3 | 多数 API 有基本签名，但缺详细参数说明和示例 |
| 2 | 仅有头文件声明，文档严重不足 |
| 1 | 大量 API 无文档，开发者基本无法使用 |
