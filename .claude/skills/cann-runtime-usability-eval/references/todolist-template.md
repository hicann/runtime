## 任务标签说明

每条任务须标注所属模块标签，从以下选项中选取：

| 标签 | 适用范围 |
|------|---------|
| `[基础设施]` | README、LICENSE、CONTRIBUTING 等仓库基础文件 |
| `[API文档]` | API Reference 相关文档 |
| `[编程模型]` | Device-Context-Stream 编程模型文档 |
| `[快速上手]` | 快速开始指南、入门教程 |
| `[示例代码]` | example/ 下的示例代码 |
| `[错误处理]` | 错误码文档、排查指引 |
| `[构建系统]` | CMake、编译脚本相关 |
| `[开发者体验]` | 综合体验改善 |
| `[生态集成]` | 与上下游组件的集成 |
| `[FAQ]` | 常见问题解答 |
| `[衔接]` | AscendC 与 Runtime 衔接缺口专用 |

## 生成规则

1. **完全从 Roadmap 提取**，须覆盖 Roadmap 所有问题项
2. **可合理拆分**：一个 Roadmap 条目如包含多个独立行动项，可拆为多条任务
3. **可合理合并**：多个 Roadmap 条目如指向同一改进动作，可合为一条任务
4. **不得遗漏**：任何 Roadmap 条目都必须能在映射追溯表中找到对应的 TodoList 任务
5. **任务描述精炼**：比 Roadmap 更简洁，突出"做什么"而非"为什么"
6. **可直接认领**：任务描述应清晰到负责人可以直接开始执行

## 进度更新规则

- 完成一项任务后，将 `- [ ]` 改为 `- [x]`
- 更新文件头的完成进度条
- 进度计算：已完成任务数 / 总任务数 × 100%
