## 新增错误码测试模板

如需新增其他错误码测试，按以下模板编写：

```cpp
// ============================================================================
// EEXXXX: 错误描述 - source_file.cc:line_number
// ============================================================================
TEST_F(ErrMsgRealTest, EEXXXX_ErrorName) {
    std::cout << "\n[Test] EEXXXX - Error Description" << std::endl;
    std::cout << "Source: source_file.cc:line_number\nMacro: COND_RETURN_AND_MSG_OUTER(...)" << std::endl;
    
    // 准备参数
    std::string param1 = "value1";
    std::string param2 = "value2";
    
    // lambda 包装宏调用
    auto test_func = [&]() -> rtError_t {
        COND_RETURN_AND_MSG_OUTER(true, RT_ERROR_XXX, ErrorCode::EEXXXX, param1, param2);
        return RT_ERROR_NONE;
    };
    rtError_t ret = test_func();
    std::cout << "[DEBUG] Macro returned: " << ret << std::endl;
    
    std::string err_msg = GetAndPrintErrMsg();
    
    // 验证 ErrMsg 内容
    EXPECT_TRUE(err_msg.find("expected_text") != std::string::npos);
    EXPECT_TRUE(err_msg.find("EEXXXX") != std::string::npos);
}
```

## Lambda 包装技巧说明

问题：`COND_RETURN_AND_MSG_OUTER` 宏内部有 `return RTERRCODE`，但 gtest 的 `TEST_F` 是 void 函数。

解决：用 lambda 包装，lambda 返回 `rtError_t`，宏的 return 在 lambda 内生效：
```cpp
auto test_func = [&]() -> rtError_t {
    COND_RETURN_AND_MSG_OUTER(true, ...);  // 宏内的 return 在这里生效
    return RT_ERROR_NONE;  // 条件不满足时返回
};
rtError_t ret = test_func();  // 调用 lambda
```
