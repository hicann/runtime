# 代码完成度统计模板

## 统计格式

```
总步骤数：12（Step 1-8 含 7a-7b 和 8a-8c，Step 9-12 后续流程）
Runtime 基础 API 步骤：N 步
aclnn 衔接区域步骤：N 步
├─ 可完成（有明确文档支撑）：N 步
├─ 靠猜测勉强完成：N 步
└─ 无法完成（资料完全缺失）：N 步
代码完成率：XX%
```

## 完成率计算

```
代码完成率 = 可完成步骤数 / 总步骤数 × 100%
```

注意：
- "靠猜测"的步骤**不计入**可完成，因为推测性代码不可靠
- 衔接区域的步骤按 Runtime 仓库文档（docs/ + example/）的覆盖情况判定
- 如果仅靠 example/ 代码反推而文档无说明，算"靠猜测"
- 如 $0 不需要 Scalar 参数，步骤 7b 标注为"不适用"并从总步骤数中扣除
